# 会议信息管理系统 — 下一阶段开发需求文档

> 编制时间：2026-07-06
> 编制人：WorkBuddy
> 参考产品：**智会行**（南京智会行数字科技有限公司 / hui-yi.com.cn / hd.jsmice.com）功能体验站
> 当前项目基础：v1 微站（hero+倒计时/议程/嘉宾/会场/直播/座位查询/报名）+ 管理后台（CRUD/CSV 导出），数据层 `localStorage`

---

## 0. 文档说明

本文档将"智会行"产品的功能矩阵拆解为可落地到我方纯前端会议信息管理系统的迭代需求，分为：

- **P0 必做**（核心闭环，与现有架构兼容，2 周内可上）
- **P1 重要**（商业化关键能力，1 个月内可上）
- **P2 增强**（差异化亮点、视精力排期）
- **P3 战略级**（需要后端 / 小程序 / 硬件，V2+ 阶段）

每条需求给出：
- **用户故事**（User Story）
- **功能要点**（Function Points）
- **数据模型变化**（Data Schema Delta）
- **验收标准**（Acceptance Criteria）
- **依赖**（Dependencies）

## 2. P0 — 必做（核心闭环 + 体验修复）

### P0-1 团队报名（多人一次性录入）
- **用户故事**：作为企业行政，我想一次性帮 5~20 位同事统一报名，避免让同事各自填表。
- **功能要点**：
  - 报名表单支持"添加成员"行内追加，最多 20 人/单
  - 支持 Excel 粘贴（姓名、手机、单位、性别字段），自动拆分多行
  - 同一团队使用同一发票抬头，生成统一订单号
  - 团队队长可分享链接让队员补填手机号
- **数据模型变化**：`regs` 数组增加 `teamId`、`teamLeader`、`invoiceTitle`、`orderNo`
- **验收标准**：可一次录入 5+ 人；团队报表可按 `teamId` 聚合导出
- **依赖**：无（localStorage 即可）

### P0-2 我的报名 / 个人中心
- **用户故事**：作为已报名的用户，我想在微站里查到自己报了什么场次、我的座位号、我的电子票。
- **功能要点**：
  - 微站个人中心 tab，输入"姓名+手机"查询
  - 展示：我的报名、座位号（含座位图高亮）、日程提醒、电子票二维码（base64 内联生成）、可下载的 PDF 票
  - 报名后系统写入 `myReg` 缓存，扫码自动回填
- **数据模型变化**：新增 `myReg`（key=姓名|手机）
- **验收标准**：报名成功后 1 步可进入个人中心，座位高亮正确
- **依赖**：座位分配模块已具备

### P0-3 邀请函生成（H5 / 海报）
- **用户故事**：作为已报名用户，我想生成分享海报发朋友圈。
- **功能要点**：
  - 个人中心"生成邀请函"按钮
  - 模板：会议主题、嘉宾姓名、二维码、座位号、会议时间地点
  - 基于 `html2canvas`（CDN）前端生成 PNG，可下载/分享
  - 主办方后台可配置多套邀请函模板
- **数据模型变化**：`conf.inviteTemplates: [{id, name, bg, fields}]`
- **验收标准**：3 步内生成 1080×1920 海报，中文不糊
- **依赖**：html2canvas 引入（约 70KB gzipped）

### P0-4 通知提醒（多通道）
- **用户故事**：作为已报名用户，我想在会前 1 天 / 1 小时收到提醒，不要错过会议。
- **功能要点**：
  - 微站浮窗 + 浏览器通知 + Service Worker 离线推送（兼容降级）
  - 提醒类型：开会议程开始、座位查询开放、议程变更、抽奖开始、直播开始
  - 用户可关闭 / 调整提前量
- **数据模型变化**：`myReg.notifOpts: {swBefore, agendaBefore, ...}`
- **验收标准**：在 PC Chrome 上至少能弹出 1 次系统通知
- **依赖**：Service Worker 静态部署即可

### P0-5 微站访问统计（PV/UV）
- **用户故事**：作为主办方，我想知道有多少人看了微站、看了哪些页面、停留多久。
- **功能要点**：
  - 页面 PV 计数 + 设备指纹粗略 UV（localStorage 自增）
  - 关键行为埋点：报名、座位查询、议程查看次数、邀请函生成次数
  - 后台新增"数据统计"tab
- **数据模型变化**：`stats: {pv, uv, pagePV:{path:count}, actions:{name:count}}`
- **验收标准**：访问 index.html 后 5 秒内可在后台看到 UV+1
- **依赖**：无

## 3. P1 — 重要（商业化关键能力）

### P1-1 智会订票（票种 + 库存 + 订单）
- **用户故事**：作为主办方，我想售卖早鸟票/标准票/VIP 票，每种票有数量上限。
- **功能要点**：
  - 后台票种管理：名称、原价、早鸟价、库存、生效时间、限购数
  - 报名时选票种、自动计算价格、生成订单号
  - 票务核销：个人中心展示"已购票"，扫码核销
  - 库存用 `localStorage` 模拟（演示级）；正式版预留后端接口
- **数据模型变化**：`tickets: [{id, name, price, earlyPrice, stock, sold, perUser, startAt, endAt}]`、`orders: [{id, ticketId, count, amount, status}]`
- **验收标准**：库存卖完后再次报名提示已售罄；订单可在后台查询
- **依赖**：P0-1 团队报名

### P1-2 智会订房（推荐酒店 + 预订）
- **用户故事**：作为外地参会者，我想直接通过微站订协议价酒店。
- **功能要点**：
  - 后台酒店管理：名称、地址、价格、房型库存、协议价、地图坐标
  - 微站酒店 tab 列表 + 关键字搜索（已有骨架：广州白云国际会议中心）
  - 选房型 → 填入住人 → 生成订房单 → 后台汇总
- **数据模型变化**：`hotels: [{id, name, addr, lat, lng, rooms:[{type, price, stock}]}]`、`bookings: [{hotelId, roomType, checkIn, checkOut, user}]`
- **验收标准**：可下单订房，后台能看到订单 + 总价
- **依赖**：无

### P1-3 智会接送（车辆信息 + 班次）
- **用户故事**：作为参会者，我想查看接机/接站班次，知道几点在哪接我。
- **功能要点**：
  - 后台班次管理：日期、出发地（机场/火车站）、目的地酒店、车型、容量
  - 微站展示"我的班次"（按姓名+手机匹配已报名信息）+ 完整班次表
  - 用户可"加入班次"占座
- **数据模型变化**：`shuttles: [{id, date, from, to, type, capacity, taken}]`
- **验收标准**：报名成功后自动按到达时间推荐班次
- **依赖**：报名时增加 `arrival`（到达时间/地点）字段

### P1-4 智会签到（前台自助 / 一体机模拟）
- **用户故事**：作为主办方，我想让参会者到场扫码 1 秒签到，代替人工查表。
- **功能要点**：
  - 后台"签到管理"：二维码大屏（每场会议一个签到码，姓名+手机输入兜底）
  - 签到后状态写 `regs[i].checkedIn = {at, gate, device}`
  - 微站个人中心显示"已签到"绿标 + 时间
  - 提供"签到墙"页（已签到头像墙，参考智会行"主K翻转"）
- **数据模型变化**：`regs[i].checkedIn`
- **验收标准**：扫码进入签到页 → 输入手机 → 显示"签到成功"+ 头像墙加入
- **依赖**：P0-2

### P1-5 资料下载 / 会议手册
- **用户故事**：作为参会者，我想下载会议手册和嘉宾 PPT。
- **功能要点**：
  - 后台上传：会议手册、嘉宾 PPT、视频回放地址、附件（PDF/ZIP）
  - 微站"资料"tab 列表 + 文件类型图标 + 大小 + 下载次数统计
  - 权限：可设"仅签到用户下载"或"全员可下载"
- **数据模型变化**：`materials: [{id, name, type, url, size, requireCheckIn, downloads}]`（已有，需扩展）
- **验收标准**：可下载 PDF 并统计次数
- **依赖**：无

### P1-6 学分管理（医学会议刚需）
- **用户故事**：作为参会医师，我想会后自动获得学分证书。
- **功能要点**：
  - 后台"学分"管理：项目编号、授予学分、考勤门槛（签到时长）
  - 微站个人中心"我的学分"，自动累计参会场次
  - 签退/退场时间记录，会后判定是否达标
  - 颁发电子学分证书（带二维码核验页）
- **数据模型变化**：`credits: {projectNo, required, sessions:[{id, hours}]}`、`myCredits: [{userKey, projectNo, hours, certUrl}]`
- **验收标准**：参会满 N 场后自动生成证书 PDF
- **依赖**：P1-4 签到

### P1-7 缴费 / 发票（演示级）
- **用户故事**：作为参会者，我想报名时直接微信支付并拿到发票。
- **功能要点**：
  - 价格字段 + 微信/支付宝支付（演示用"模拟支付"按钮）
  - 后台"缴费"tab：订单、金额、状态、申请人
  - 后台"发票"tab：抬头、税号、邮箱、状态（待开/已开）
  - 真实支付需对接第三方，预留接口
- **数据模型变化**：`orders`、`invoices: [{id, userKey, title, taxNo, email, status}]`
- **验收标准**：能跑通"下单→支付→开票"全流程
- **依赖**：P1-1 票务

## 4. P2 — 增强（差异化亮点）

### P2-1 互动抽奖（年会场景）
- 大屏抽奖墙：滚动姓名 / 头像 / 座位号，支持 3D 老虎机动画
- 奖项配置：一/二/三等奖、人数、奖品
- 中奖名单同步到微站个人中心
- 复用 P0-2 `myReg` 头像库

### P2-2 圆桌查询（晚宴座位）
- 现有座位查询是"剧场式"（A/B/C 区排号），新增**圆桌式**模式：
  - 每桌 8/10 人，输入姓名查桌号
  - 餐桌平面图，缩放、查看同桌人
  - 后台"排桌"：手动拖拽 / 一键按团队 / 按 VIP 自动分配
- 字段：`seatsRound: [{tableNo, capacity, members:[userKey]}]`

### P2-3 课桌查询（培训/考试）
- 仿课桌式排布：长条桌 + 椅号
- 后台"排课桌"：考号 / 考生姓名绑定
- 适用培训签到、考场座位

### P2-4 劳务系统（志愿者/工作人员调度）
- 工作人员花名册：姓名、手机、岗位、所属组（接待/会场/物料/茶歇）
- 后台"打卡"：上岗/离岗时间
- 微站"工作人员端"：查看自己的排班 + 扫码打卡
- 字段：`staff: [{id, name, phone, role, group, shifts:[]}]`

### P2-5 专家系统（讲者/主持聚合页）
- 主页"专家"tab：按场次/分会/单位筛选
- 讲者详情页：头像、简介、所属单位、所在场次、议程时间
- 已有嘉宾管理可复用，扩展 `expertSessions: []`

### P2-6 3D 云展厅 / 线上展厅
- 后台上传展位平面图（图片）→ 前端渲染可缩放 canvas
- 每个展位挂资料 + 视频 + 链接
- 第三方方案：360vryun 嵌入 / 自主 canvas 实现
- 字段：`exhibitors: [{id, booth, name, logo, materials, videoUrl}]`

### P2-7 电子壁报
- 上传 PDF/图片作为"壁报"，在线翻页（pdf.js）
- 投票 / 收藏 / 评论（演示级）
- 字段：`posters: [{id, title, file, votes, favorites}]`

### P2-8 视频直播 / 图片直播
- 视频：嵌入腾讯会议/保利威/微赞直播间 iframe
- 图片直播：嵌入 yipai360 / 自建图床相册
- 微站"直播"页带倒计时"距离开播还有…"
- 字段：`streams: [{id, title, type, url, startAt}]`

### P2-9 海报合成（邀请函升级版）
- 模板编辑器：拖拽字段、画布（Konva.js）
- 批量生成（团队报名后一键生成 N 张成员海报）
- 已有 P0-3 基础版

### P2-10 地图点亮 / 主 K 翻转
- 大屏特效：参会者签到时把头像投到全国/区域地图，签到满后整图点亮
- 主K翻转：所有签到头像拼成会议主视觉
- 复用 P1-4 签到头像数据

### P2-11 议程"我的日程"个性化
- 议程页支持"收藏"按钮
- 个人中心展示"我的日程"：按时间排序、ICS 日历导出
- 字段：`myReg.favAgenda: []`

### P2-12 多语言（中/英）
- 议程双语：议程条目 `titleEn`、嘉宾 `nameEn`、会场介绍 `descEn`
- 顶部语言切换
- 国际会议刚需

## 5. P3 — 战略级（V2+ / 需后端 / 硬件）

### P3-1 智会签到一体机（硬件）
- 桌面安卓一体机 + 二维码扫描头 + 摄像头
- 离线模式：本地 SQLite 缓存 + 联网后同步
- 暂用 PC 浏览器模拟开发，预留 REST 接口

### P3-2 智会电子席卡（硬件）
- 蓝牙/e-ink 桌牌，扫码绑定座位
- 改座后自动更新
- 暂用 PC 浏览器模拟开发

### P3-3 小程序端（微信生态扩展）
- 复用微站数据模型，迁移到原生小程序
- 微信"订阅消息"代替浏览器通知
- 共享后台 API

### P3-4 后端 SaaS 化
- NestJS + PostgreSQL + Redis
- 多租户（一个机构一个 workspace）
- RBAC 权限（超级管理员 / 主办方 / 工作人员 / 嘉宾 / 参会者）
- 真实支付、对账、电子发票
- 数据看板：实时签到数、参会画像、热力图

### P3-5 数字人 / AI 客服
- 会议 AI 助手：问答、议程推荐、找座位、找洗手间
- 数字人主持/播报
- 需 LLM 接入

### P3-6 直播带货 / 招商
- 微站挂赞助商展位 + 直播带货
- 付费曝光位

## 7. 数据模型 v2 草案（顶层）

```js
window.ConfStore = {
  // 现有 v1
  conf, speakers, agenda, regs, seats, materials,

  // v2 新增
  tickets, orders, hotels, bookings, shuttles,
  myReg, myCredits, invoices,
  staff, shifts, checkIns,
  exhibitors, posters, streams,
  stats, notifs,
  inviteTemplates, creditConfig
}
```

> 全部仍走 `localStorage`，单 KEY 容量上限 5MB（够用）；超过则提示清理历史数据。生产环境预留 API 切换层 `window.ConfStore.read/write` 抽象。

## 9. 风险与决策点

- **数据持久化**：v1.x 继续 localStorage；v2.x 切换为后端 API 时需设计 `ConfStore` 抽象层，向后兼容。
- **支付 / 发票**：纯前端做不到真支付，v1.x 用"模拟支付"占位 + 标记 TODO，v3.0 接微信/支付宝。
- **直播 / 3D 展厅**：不自研视频流，直接嵌入第三方 iframe（保利威、微赞、360vryun、yipai360）。
- **微信兼容**：微站用 H5，不上小程序；个人中心"我的报名"用 URL 参数（`?uid=`）免登录，扫码即看。
- **多端同步**：v3.0 上线前不做，服务端化解决。
